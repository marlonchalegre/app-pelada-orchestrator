# Financial Control Feature - Development Handover

This document summarizes the development of the Financial Control feature as of March 28, 2026.

## 1. Objective
Create a comprehensive financial management system within the Pelada App to track organization cash flow, "mensalista" monthly fees, and "diarista/convidado" per-game payments.

## 2. Current Implementation State

### Database Schema
Three new tables were added via migrations:
- **`OrganizationFinances`**: Stores settings per organization.
    - `mensalista_price` (DECIMAL)
    - `diarista_price` (DECIMAL)
    - `currency` (VARCHAR)
- **`Transactions`**: The immutable (audit-trail) ledger.
    - `type`: 'income' or 'expense'
    - `category`: 'monthly_fee', 'diarista_fee', 'field_rent', etc.
    - `amount`: Monetary value (supports decimals).
    - `status`: 'paid' (included in calculations) or 'reversed' (excluded).
    - `created_by`: ID of the admin who recorded it.
- **`MonthlyPayments`**: Tracks monthly status for mensalistas.

### Backend API (`api-peladaapp`)
- **Immutability Logic**: Transactions are not deleted. Chargebacks (reversals) trigger a `reverse-transaction` call which sets the status to `'reversed'`.
- **Calculations (Verified)**: `get-summary` calculates Total Income, Total Expense, and Balance by filtering `WHERE status = 'paid'`.
    - **Income Reversal**: Reversing a positive transaction (income) correctly deducts from "Total Income" and "Balance".
    - **Expense Reversal**: Reversing a negative transaction (expense) correctly deducts from "Total Expense" and increases "Balance".
- **Pagination**: `list-transactions` now supports pagination (`page` and `per_page` query params) and returns `X-Total` headers.
- **Authorization**: All `/finance` endpoints are restricted to organization administrators using `auth/require-organization-admin!`.
- **User "Me" Endpoint**: Added `GET /api/users/me` to support reliable session refreshing and admin permission checking.

### Frontend (`web-peladaapp`)
- **`FinanceSection` Component**: A new tab in Organization Management with:
    - **Summary**: Real-time balance cards.
    - **Monthly Fees**: Grid to mark mensalistas as paid/unpaid.
    - **Transactions**: A paginated ledger showing date, description, category, value, and the admin responsible.
    - **Configuration**: Form to set monetary values (supports float/decimal input via local string state).
- **Dashboard Integrations**: Per-game payment icons added to:
    - Attendance List
    - Player Selection Panel (Bench)
    - Active Match Dashboard
- **Reversal UI**: Reversed transactions appear grayed out with a strikethrough.

## 3. Usage Notes
- **Marking Paid**: Creates a `paid` transaction -> Income/Expense increases, Balance updated.
- **Unmarking Paid (Reversal)**: Marks the *existing* transaction as `reversed` -> Income/Expense correctly deducted, Balance updated.
- **Floating Point**: Inputs use a text-based parser to allow smooth typing of decimal points/commas.

## 4. Verified Scenarios
- **Scenario 1**: Start (0,0,0) -> Deposit 10 -> Result (10,10,0) -> Reverse -> Result (0,0,0). Correct.
- **Scenario 2**: Start (0,0,0) -> Buy Ball 100 -> Result (-100,0,100) -> Reverse -> Result (0,0,0). Correct.
